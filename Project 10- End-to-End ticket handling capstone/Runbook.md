# Project 10: End-to-End Ticket Handling Capstone

## Objective
Demonstrate job readiness across identity, security, and compliance by resolving common support tickets in an identity-only environment.

## Prerequisites
- Microsoft Entra ID access
- Test users (TestUser1, TestUser2)
- Security groups (App-Access-Users, App-Access-Admins)
- Basic understanding of MFA and sign-in logs

---

## Ticket 1 – Access Request

I reviewed an access request and validated the user’s role. I assigned TestUser1 to the appropriate security group following least privilege access.

**Outcome:** Access granted using group-based control.

---

## Ticket 2 – MFA Issue

I reviewed the user’s authentication methods and confirmed MFA was configured correctly. I ensured the user had valid MFA methods registered.

**Outcome:** MFA configuration verified.

---

## Ticket 3 – Security Escalation

I reviewed sign-in logs for suspicious activity and followed escalation procedures based on observed sign-in behavior.

**Outcome:** Security activity reviewed and documented.
